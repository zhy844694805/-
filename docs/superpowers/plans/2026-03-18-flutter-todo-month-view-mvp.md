# Flutter Todo Month View MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an iOS/Android Flutter todo app MVP with Magic Link login, Firestore-backed sync, offline-capable month view, and a day-detail task sheet.

**Architecture:** Start from a fresh Flutter mobile app in this repository, then layer Riverpod-managed application state on top of repository interfaces so widget and integration tests can swap Firebase implementations for fakes. Use a custom month grid UI backed by a `dueDateKey` (`YYYY-MM-DD`) query model in Cloud Firestore so month aggregation, offline cache, and day selection remain deterministic.

**Tech Stack:** Flutter, Dart, flutter_riverpod, go_router, firebase_core, firebase_auth, cloud_firestore, app_links, shared_preferences, connectivity_plus, intl, fake_cloud_firestore, mocktail, firebase_auth_mocks, integration_test

class LocalDate {
  const LocalDate(this.year, this.month, this.day);

  final int year;
  final int month;
  final int day;

  String get dueDateKey =>
      '$year-${month.toString().padLeft(2, '0')}-${day.toString().padLeft(2, '0')}';

  DateTime get asDateTime => DateTime(year, month, day);

  static LocalDate today() {
    final now = DateTime.now();
    return LocalDate(now.year, now.month, now.day);
  }

  @override
  bool operator ==(Object other) =>
      other is LocalDate &&
      other.year == year &&
      other.month == month &&
      other.day == day;

  @override
  int get hashCode => Object.hash(year, month, day);
}

- [ ] **Step 3: Implement the Firestore repositories**

Use contracts shaped like this:

```dart
abstract class TaskRepository {
  Stream<List<TaskItem>> watchMonth(LocalDate month);
  Future<void> upsert(TaskItem task);
  Future<void> toggleCompleted({
    required String taskId,
    required bool isCompleted,
  });
}

class FirestoreTaskRepository implements TaskRepository {
  FirestoreTaskRepository({required this.firestore, required this.userId});

  final FirebaseFirestore firestore;
  final String userId;

  @override
  Stream<List<TaskItem>> watchMonth(LocalDate month) {
    final first = DateTime(month.year, month.month, 1);
    final last = DateTime(month.year, month.month + 1, 0);

    return firestore
        .collection('users')
        .doc(userId)
        .collection('tasks')
        .where('dueDateKey', isGreaterThanOrEqualTo: LocalDate(first.year, first.month, first.day).dueDateKey)
        .where('dueDateKey', isLessThanOrEqualTo: LocalDate(last.year, last.month, last.day).dueDateKey)
        .snapshots()
        .map((snapshot) => snapshot.docs.map(TaskDto.fromFirestore).map((dto) => dto.toDomain()).toList()
          ..sort((a, b) => a.dueDate.dueDateKey.compareTo(b.dueDate.dueDateKey)));
  }
}
```

Store list docs under `users/{userId}/lists/{listId}` and task docs under `users/{userId}/tasks/{taskId}`.
